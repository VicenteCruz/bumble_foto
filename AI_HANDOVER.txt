========================================================================
BUMBLE FOTO - AI CONTEXT & HANDOVER DOCUMENTATION
========================================================================
Este ficheiro destina-se a fornecer contexto rápido a assistentes de 
Inteligência Artificial que trabalhem neste repositório no futuro, 
poupando tokens e leitura exaustiva do código-fonte original.

[ 1. ARQUITETURA GERAL ]
- Projeto: "Bumble Foto", uma aplicação web "serverless" (100% frontend).
- Objetivo: Fazer triagem rápida de grandes pastas de fotografias locais
  usando gestos de swipe estilo Tinder (Esquerda = Apagar, Direita = Manter,
  Baixo = Agrupar semelhantes).
- Tecnologia: HTML5, CSS3, Vanilla JavaScript (ES6+). Sem frameworks, sem backend.
- API Core: "File System Access API" nativa do browser (Chrome/Edge/Opera + HTTPS/localhost).
  -> A app pede permissão de leitura/escrita de uma diretoria local.
  -> Move ficheiros para as subpastas `_kept`, `_trash` e `_groups`.

[ 2. ESTRUTURA DE FICHEIROS ]

1. index.html
   - Documento principal. Organizado por "Screens" alternados via CSS (`.active`).
   - `#folder-screen`: Ecrã inicial com `showDirectoryPicker()`.
   - `#swipe-screen`: Ecrã de triagem com `#photo-card` e previews.
   - `#done-screen`: Ecrã final de estatísticas.
   - `#stats-screen`: Dashboard com mapa de calor (Leaflet) e gráficos (Chart.js).
   - Modais: `#settings-modal`, `#group-modal`, `#gallery-modal`, `#groups-list-modal`.
   - NOTA: Não há inline event handlers (onclick) — tudo registado via addEventListener no JS.
   - NOTA: Estilos inline minimizados — usar classes CSS do styles.css.

2. styles.css
   - Variáveis CSS nativas (`:root`) para tema escuro premium.
   - Animações de cartões: `.entering`, `.exit-left`, `.exit-right`, `.exit-group`.
   - Classes utilitárias para componentes reutilizáveis (`.btn-stats-swipe`,
     `.btn-modal-action`, `.group-modal-actions`, etc.)
   - GOTCHA: Modais usam `pointer-events: none` na base e `pointer-events: auto`
     quando `.active`. NUNCA usar `pointer-events: all` (só válido em SVG).

3. analyzer.js
   - Processamento de Imagem (Canvas escondido).
   - "Quality Tags": Deteta "Desfocada" (Laplacian), "Escura", "Estourada".
   - "Similarity": `computeDHash` (64-bit hash) + `computeColorHistogram` (64 bins).
   - Lê parâmetros globais de `window.AppSettings` (backward compat).

4. app.js
   - Coração do projeto. Controla DOM, gere pastas, gere estado.
   - Objeto `state` central:
     -> `photos`: Array de imagens pendentes.
     -> `currentIndex`: Índice da foto atual.
     -> `group`: Array temporário de fotos agrupadas.
     -> `isGrouping`: Se o modo de agrupamento está ativo.
     -> `undoStack`: Pilha de ações para desfazer.
     -> `settings`: Configurações do algoritmo (persistidas em localStorage).
   - Settings: Guardadas em `state.settings` com backward compat `window.AppSettings`.
   - `computeSimilarity(img1, img2)`: Função reutilizável para comparar similaridade
     estrutural (dHash) e de cor (histograma) entre duas imagens.
   - Gestão de Memória: Usa `URL.createObjectURL()` — CRÍTICO fazer sempre
     `URL.revokeObjectURL()` ao trocar imagens.
   - Preview layout: Gerido por `state._updatePreviewsLayout` (closure no setupEventListeners).

[ 3. LÓGICA DE AGRUPAMENTO ]
- `state.isGrouping = true` quando o utilizador inicia um grupo (tecla M).
- Fotos adicionadas a `state.group`, UI mostra preview "Anterior" com moldura amarela.
- Detecção automática de similaridade via `computeSimilarity()` — marca com `.current-group-alert`.
- `endGrouping()` abre modal para o utilizador escolher quais manter/apagar/decidir mais tarde.
